Se me ocurre que:

1-  Encripte las contraseñas mediante gpg, para que con la llave de ella
pueda desencriptar la información (Esto da más seguridad en caso de que
otra persona no autorizada tenga acceso al archivo y lo quieran abrir ).

2- Realizar un script o programa y ponerlo en un sitio seguro (servicio de
hosting) para que cada determinado tiempo solicite al usuario una contraseña que solo
el usuario sepa para indicarle al script que sigue vivo.

Descripcion del funcionamiento del script o programa:
Se necesita un script que cada cierto tiempo le solicite al usuario una
contraseña y otros mecanimos de verificacion como pueden ser otras API:

Si la contraseña es ingresada: el script no hace nada más que esperar hasta
que el plazo de pedir la contraseña nuevamente se cumpla.

Si la contraseña no fue ingresada: el script puede tener otros mecanismos para
verificar si el usuario sigue vivo:

Lista de alternativas y mecanismos para la verificación:

1-Prolongar el plazo y contactar al usuario por medio de otra persona (un
tercer contacto), el script podria enviar un correo a otro miembro de la familia del
usuario para que le avise que el plazo se ha cumplido y que si sigue vivo el
usuario le recuerde realizar la tarea de autentificacion (ya que puede darse el caso que al
usuario se le olvide o ande de vacaciones o sin acceso a internet etc).

2-Comprobar con alguna otra API la ultima fecha de inicio de sesion del
usuario (con esto se puede verificar si el usuario sigue vivo y ha logeado
ultimamente en algun otro servicio online) si el usuario no ha hecho uso de algun
otro servicio en los ultimos 2 meses por ejemplo entonces lo más probable es
que el usuario este muerto.

3-Instalar extensiones del script por ejemplo en un cron deamon para que cada
que logie el usuario a su maquina automaticamente le avise al script principal
que el usuario sigue vivo y logeando en su maquina personal o dispositivo
movil.

Despues de que todas o la mayoria de comprobaciones que realice el script
indiquen de que es muy probable de que el usuario este muerto, entonces envie
el archivo encriptado al correo de su esposa para que ella con la contraseña
pueda abrirlo.

Aun así con la idea anterior planteada, podria recomendar una caja fuerte en
su habitación con las contraseñas u otro material que quiera entregar a su
esposa despues de morir, en el peor de los casos que el script falle o ambos
mueran la caja fuerte puede ser forzada fisicamente para recuperar la
información.

Contratar un servicio de caja fuerte en un banco como se ha visto en las
peliculas tambien puede ser buena idea si se tiene dinero y las ganas de pagar
un servicio así para unas contraseñas.
